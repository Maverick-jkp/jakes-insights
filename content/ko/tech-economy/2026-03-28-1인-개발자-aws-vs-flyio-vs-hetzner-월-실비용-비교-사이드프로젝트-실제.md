사이드프로젝트 첫 달 AWS 청구서 받고 "이게 뭐야" 했던 경험, 맞죠?

## 1. 지금 이 비교가 필요한 이유

Micro SaaS 붐이 이어지면서 혼자 서비스 만들고, 운영하고, 수익화까지 하는 시도가 늘고 있어요. 문제는 인프라 비용이에요.

AWS는 한국 개발자들에게 사실상 표준처럼 자리 잡았지만, 첫 청구서 받기 전까지 얼마가 나올지 아는 사람이 거의 없어요. "공식 요금표엔 싸 보였는데 왜 이렇게 나왔지?" 반응이 나오는 건 이유가 있어요. 데이터 전송료, NAT Gateway, CloudWatch 로그 스토리지가 조용히 쌓이거든요.

반면 Fly.io는 2023년부터 인디 해커 커뮤니티에서 빠르게 주목받았고, Hetzner는 유럽 기반 1인 개발자들 사이에서 "가성비 끝판왕"으로 통해요. r/Hosting에서도 "Hetzner 쓰면 AWS 대비 절반 이하 비용"이라는 실사례가 꾸준히 올라오고 있고요.

이 글에서 다룰 포인트는 네 가지예요.

- 트래픽 적은 사이드프로젝트 기준 실제 월 청구 금액 비교
- AWS 숨은 비용 항목 해부
- Fly.io vs Hetzner 실사용 트레이드오프
- 프로젝트 단계별 어디서 시작해야 하는지 결론

> **핵심 요약**
> - AWS는 t3.micro 1대만 써도 데이터 전송료·NAT Gateway 등 숨은 비용으로 월 30~60달러가 추가되는 사례가 커뮤니티에서 반복 보고되고 있어요.
> - Fly.io 무료 플랜은 공유 CPU 3개, 3GB 메모리, 160GB 스토리지를 제공하지만, 2024년 말 유료 전환 정책 변경 이후 실질적인 무료 한도가 줄었어요.
> - Hetzner CX22 기준 월 4.15유로(약 6,000원)로 2vCPU·4GB RAM 서버를 운영할 수 있어요.
> - 단일 정답은 없고, 트래픽 규모와 관리 여력에 따라 최적 선택지가 달라져요.

## 2. 1인 개발자는 왜 인프라 비용에 민감한가

이유는 단순해요. 월 매출 50만 원인 서비스에 인프라 비용이 30만 원이면 남는 게 없거든요.

2022~2023년 AWS 프리 티어 믿고 시작했다가, 12개월 만료 후 급등한 청구서를 받는 경험담이 Reddit과 인디해커 포럼에서 꾸준히 올라왔어요. AWS 공식 문서 기준, 프리 티어 만료 후 EC2 t3.micro는 서울 리전 기준 시간당 약 0.0116달러예요. 한 달이면 약 8.5달러. 얼핏 싸 보이죠?

문제는 EC2 비용이 전부가 아니라는 거예요.

- **데이터 전송 비용**: 인터넷으로 나가는 트래픽은 월 1GB까지만 무료, 이후 GB당 0.09달러
- **NAT Gateway**: 시간당 0.059달러 고정 + 데이터 처리 GB당 0.059달러
- **RDS, CloudWatch, S3**: 각각 별도 과금

Hetzner는 2019년부터 가성비로 유럽 개발자 커뮤니티에서 인기를 끌기 시작했고, Fly.io는 2021년 등장해 컨테이너 기반의 단순 배포 경험으로 인디 해커들에게 어필했어요.

## 3. 실비용 분석: 세 플랫폼을 직접 뜯어보면

### AWS: 예측 불가능한 청구서의 구조

AWS는 레고예요. 필요한 부품을 조합해서 쓸 수 있지만, 부품 하나하나가 다 돈이에요.

사이드프로젝트 기본 구성(EC2 t3.small, RDS t3.micro, ALB, 데이터 전송 10GB/월)으로 서울 리전 예상 월 비용을 AWS 공식 Pricing Calculator로 계산하면 대략 **65~90달러** 선이에요. ALB만 빼도 20달러 이상 절감되지만, SSL 관리를 직접 해야 해요.

커뮤니티에서 공유된 실제 청구 패턴을 보면, 예상치 못한 비용의 주범 1위는 NAT Gateway예요. VPC 기본 설정으로 시작했을 때 NAT Gateway를 모르고 켜두면 월 40달러 이상이 나오는 경우도 있어요.

### Fly.io: 배포 경험은 좋지만 프리 티어는 줄었어요

Fly.io의 강점은 `fly deploy` 한 줄이에요. Docker 이미지만 있으면 전 세계 어느 리전에도 30초 만에 배포돼요. 2024년 말 유료 플랜 개편 이후 새 계정은 신용카드 등록 없이 쓸 수 있는 범위가 좁아졌어요.

2026년 3월 기준 shared-cpu-1x (256MB RAM) 기준 월 약 **1.94달러**부터 시작해요. 실제 웹앱 하나를 Postgres 포함해서 운영하면 월 **15~25달러** 수준이 일반적이에요. 트래픽이 몰릴 경우 자동 스케일링으로 비용이 올라가는 구조라, 상한선을 설정하지 않으면 깜짝 청구서를 받을 수 있어요.

### Hetzner: 가격 기준점

Hetzner의 포지션은 명확해요. 예측 가능한 고정 비용.

| 플랜 | vCPU | RAM | 스토리지 | 트래픽 | 월 비용 |
|------|------|-----|----------|--------|---------|
| CX22 | 2 | 4GB | 40GB SSD | 20TB | €4.15 |
| CX32 | 4 | 8GB | 80GB SSD | 20TB | €8.22 |
| CX42 | 8 | 16GB | 160GB SSD | 20TB | €16.90 |

CX22 기준 월 약 6,000원이에요. AWS t3.small 하나 값으로 CX42를 쓸 수 있어요. 단, 서버 직접 관리(Ubuntu 설정, Nginx, SSL, 백업)를 본인이 해야 해요.

r/Hosting 커뮤니티에서도 "마이크로서비스 운영하는 솔로 개발자라면 Hetzner가 비용 대비 가장 낫다"는 의견이 다수였어요. 다만 서울 리전이 없어서 한국 사용자 대상 서비스라면 레이턴시가 고려 대상이에요.

### 세 플랫폼 종합 비교

| 항목 | AWS | Fly.io | Hetzner |
|------|-----|--------|---------|
| 기본 월 비용 (소규모) | $65~90 | $15~25 | $5~10 |
| 숨은 비용 위험 | 높음 | 중간 | 낮음 |
| 배포 편의성 | 복잡 | 매우 쉬움 | 중간 |
| 한국 리전 | ✅ 서울 | ✅ 도쿄 | ❌ 없음 |
| 관리 부담 | 낮음 | 낮음 | 높음 |
| 적합한 단계 | 성장기·팀 단위 | MVP·초기 런칭 | 안정화 단계 |

트레이드오프가 명확하죠. AWS는 비싸지만 모든 게 관리형이에요. Fly.io는 배포가 빠르고 초기 비용이 적절하지만, 트래픽 급증 시 비용 예측이 어려워요. Hetzner는 가격이 압도적이지만 VPS 관리 경험이 없으면 초기 진입 장벽이 있어요.

## 4. 프로젝트 단계별 어디서 시작할까

**시나리오 1: 아이디어 검증 단계 (MAU 0~500명)**

Fly.io가 맞아요. `fly launch` 하나로 시작하고, Fly Postgres 붙이고, 도메인 연결까지 한 시간 안에 끝나요. 월 15~20달러로 아이디어 검증 충분히 돼요. AWS 프리 티어로 시작하면 12개월 후 갑자기 비용이 올라가는 함정이 있어요.

**시나리오 2: 안정화 단계 (MAU 500~5,000명, 수익 발생)**

Hetzner CX32 + Coolify 조합을 고려해볼 만해요. Coolify는 오픈소스로 Heroku 수준의 배포 경험을 Hetzner 위에서 구현해줘요. 월 10달러 미만으로 Fly.io보다 훨씬 많은 리소스를 쓸 수 있어요. 단, 한국 사용자가 주 타겟이면 도쿄 Fly.io나 AWS 서울 리전을 유지하는 게 레이턴시 면에서 나아요.

**시나리오 3: 성장 단계 (팀 합류, 규정 준수 필요)**

AWS로 가는 게 맞아요. SOC 2, GDPR, HIPAA 같은 컴플라이언스가 필요하거나, 팀이 3명 이상 붙어서 관리 부담을 나눌 수 있을 때요. 이 단계에선 AWS 관리형 서비스 생태계가 주는 시간 절약이 비용보다 크거든요.

**그리고 주시할 신호:**
- Hetzner 아시아 리전 추가 시점 (현재 공식 발표 없음)
- Fly.io 가격 정책 재개편 가능성 (2024년 변경 이후 안정화 중)
- AWS 서울 리전 데이터 전송 요금 인하 여부

## 5. 결론: 청구서보다 먼저 봐야 할 것

세 플랫폼을 정리하면 이렇게 돼요.

- **AWS**: 기능 가장 많지만, 사이드프로젝트 초기엔 비용 구조가 복잡하고 예측이 어려워요
- **Fly.io**: 가장 빠른 배포, 합리적인 초기 비용. 트래픽 급증 시 모니터링은 필수예요
- **Hetzner**: 고정 비용 최저. 직접 관리 부담과 한국 리전 부재가 트레이드오프예요

결국 1인 개발자에게 가장 비싼 인프라는 잘못 선택한 인프라예요. 지금 운영 중인 사이드프로젝트 청구서, 마지막으로 뜯어본 게 언제예요?
