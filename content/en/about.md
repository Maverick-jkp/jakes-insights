# About Jake's Insights

**Jake's Insights** is a bilingual technology publication covering AI, developer tools, cloud infrastructure, and cybersecurity — written for engineers who want analysis, not headlines.

---

## The Author

**Jake Park** is a software developer based in Seoul, Korea, currently working in the gaming and developer-tools space. He has been building production software since 2014 across backend services, content pipelines, and developer infrastructure.

Jake started this publication because most tech writing for Korean developers is either marketing-flavored translations or out-of-date. The goal is the opposite: data-first analysis, cited sources, and the same article quality for Korean readers as for English ones.

## Editorial Approach

**Community-sourced topics.** Topics are surfaced from developer communities (HackerNews, Dev.to, Lobsters, Korean dev forums) — what working engineers actually discuss, not what PR agencies pitch.

**Data-backed claims.** Cost figures, benchmark numbers, and statistics cite their source inline. When data isn't available, the article says so rather than fabricating numbers.

**No fluff.** Articles target 900–1,800 words. Long enough to cover the topic with evidence, short enough to read in a coffee break.

**Bilingual, not translated.** Korean articles are written for Korean developers in Korean idiom, not machine-translated from English copy.

## AI Content Disclosure

We use AI tooling (Anthropic's Claude API) as part of the writing pipeline. Specifically:

- **Topic selection**: Community feeds are mined and ranked algorithmically; final priority is reviewed before publication.
- **Drafting**: First drafts are generated from a prompt template that includes the topic, retrieved sources, and structural requirements.
- **Editing**: A second AI pass enforces tone, removes AI-detection phrases, and verifies citation format.
- **Quality gate**: Every draft must pass automated checks for word count, citation density, structural elements (e.g., comparison tables for "vs" posts), and AI-phrase blacklist before publishing.
- **What is *not* AI-generated**: This About page, the privacy policy, the editorial standards document, and the contact page are human-written.

If you spot a factual error — a wrong benchmark number, an outdated price, a misattributed quote — please [report it](/contact/) and it will be corrected with the original publish date preserved and an "Updated" note added.

## Editorial Standards

1. **Citations are mandatory** for any specific number (prices, percentages, benchmark results, version numbers).
2. **Real names only** for company examples. No invented "One Silicon Valley startup..." anecdotes.
3. **Failure modes are required** for comparison posts. If we can't name a real failure case for both sides, the post doesn't ship.
4. **Errors are corrected publicly** — the post gets an "Updated YYYY-MM-DD" line at the top with what changed.
