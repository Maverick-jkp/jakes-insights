Zed를 쓰다가 어느 순간 이런 생각 해본 적 없나요?

"나 AI 쓸 생각 없는데, 왜 자꾸 Agent Panel이 눈에 밟히지?"

그 불만을 코드로 해결한 사람들이 있어요. GRAM 에디터—Zed를 포크해서 AI 기능을 전부 걷어낸 오픈소스 프로젝트예요. AI 코드 에디터가 쏟아지는 2026년에, 반대 방향으로 달리는 이 프로젝트가 개발자 커뮤니티에서 조용히 퍼지고 있어요.

> **핵심 요약**
> - GRAM 에디터는 Rust 기반 Zed를 포크해 AI Agent Panel을 완전 제거한 오픈소스 에디터로, 2026년 기준 GitHub 스타가 빠른 속도로 증가하고 있어요.
> - Zed의 AI 플랜은 월 $20에 500 프롬프트를 제공하지만, GRAM은 로그인도 결제도 없이 에디터 본연의 속도만 제공해요.
> - AI 없는 Zed 포크의 핵심은 "속도 + 프라이버시 + 비용 제로"라는 세 가지 요구가 겹친 결과예요.
> - GPU 셰이더 기반 렌더링(GPUI)은 AI 없이도 VSCode 대비 체감 속도 차이가 뚜렷한데, 이 점이 GRAM을 단순 반감 프로젝트가 아닌 실용 도구로 만들고 있어요.
> - 2026년 하반기 Zed Windows 정식 출시가 예정된 상황에서, GRAM 같은 포크의 포지션이 더 선명해질 전망이에요.

## Zed가 먼저 뭔지 알아야 해요

Zed는 전체 스택을 Rust로 만든 오픈소스 코드 에디터예요. 여기서 핵심은 Electron을 쓰지 않는다는 거예요.

Electron이 뭐냐고요? VSCode, Cursor, 거의 모든 웹 기반 에디터의 엔진이에요. Chromium 브라우저 위에서 돌아가기 때문에 메모리를 많이 잡아먹고 무거워요. Zed는 그 대신 GPU 셰이더와 OS 그래픽 API를 직접 써요. GPUI라는 자체 프레임워크로요.

결과는 단순해요. 빨라요. 체감으로도요.

그런데 Zed는 2025년 들어 AI Agent Panel을 전면에 내세우기 시작했어요. Claude 3.7, Gemini 2.5를 붙이고, MCP(Model Context Protocol)로 브라우저 자동화·DB 접근·PR 생성까지 지원하고요. AI 기능이 에디터의 거의 전면에 배치돼 있어요.

문제는 이 AI 기능이 선택이 아닌 기본값처럼 자리를 차지하기 시작했다는 거예요. 월 50 프롬프트 무료, 500 프롬프트에 월 $20. 자체 API 키를 쓰면 Zed 추가 비용은 없지만, 로그인은 필요하고 구조 자체가 AI 중심으로 개편됐죠.

그때 일부 개발자들이 Zed 코드베이스를 포크해서 AI 레이어를 전부 걷어내기 시작했어요. GRAM 에디터가 그 대표적 결과물이에요.

## AI를 굳이 제거하는 이유 세 가지

**프라이버시: "기본 저장 안 됨"을 믿지 않는 사람들**

Zed는 기본적으로 Agent 대화를 서버에 저장하지 않아요. 이 정책은 사실 꽤 투명한 편이에요. 그런데 기업 환경에서는 "기본 저장 안 됨"도 충분하지 않아요. 코드가 AI 모델에 전송되는 구조 자체가 보안 심사에서 걸리는 거예요. 금융, 의료, 방산 계열 개발자들에게는 AI 기능이 아예 없는 에디터가 더 안전한 선택이에요. GRAM은 그 구조적 리스크를 애초에 없앤 에디터예요.

**비용: $0과 $20의 심리적 거리**

월 $20은 작은 돈처럼 보여요. 그런데 10명 팀이면 월 $200, 연 $2,400이에요. AI 자동완성을 즐겨 쓰지 않는 개발자라면 순수하게 낭비로 느껴지죠. GRAM은 로그인도 없고 플랜도 없어요. 설치하고 쓰면 끝이에요.

**속도: AI 레이어가 빠른 에디터를 느리게 만든다**

Rust + GPUI 기반 에디터의 장점은 렌더링 속도예요. 그런데 AI Agent Panel이 붙으면서 초기 로딩과 특정 연산에서 체감 지연이 생긴다는 리포트가 커뮤니티에 쌓이기 시작했어요. GRAM은 Zed의 핵심 엔진만 남기고 AI 관련 의존성을 제거했기 때문에 이론적으로 Zed보다 가볍게 동작해요.

## GRAM vs Zed vs VSCode

| 항목 | GRAM (Zed 포크) | Zed | VSCode |
|------|----------------|-----|--------|
| AI 기능 | 없음 | Agent Panel (Claude 3.7, Gemini 2.5) | Copilot (별도 구독) |
| 렌더링 | Rust + GPUI | Rust + GPUI | Electron |
| 월 비용 | $0 | $0~$20 (AI 사용 시) | $0~$10+ (Copilot) |
| 로그인 필요 | 없음 | AI 쓸 때 필요 | 선택사항 |
| 프라이버시 | 완전 로컬 | 기본 비저장 (전송 구조 존재) | 기본 비저장 (Copilot 별도) |
| MCP 지원 | 없음 | 지원 (브라우저, DB, PR) | 확장 플러그인 |
| Windows | 미정 | 2026 하반기 정식 예정 | 완전 지원 |
| 대상 | AI 불필요 개발자, 보안 환경 | AI 워크플로 선호 개발자 | 범용, 생태계 의존 |

세 에디터를 놓고 보면 포지션이 선명해요. VSCode는 생태계가 강점이에요. Zed는 속도와 AI를 동시에 잡으려는 에디터고요. GRAM은 그 사이 갭을 파고든 거예요. Zed의 속도 코어만 가져오고 나머지는 다 쳐냈어요.

## 실제로 누가 써야 할까요

AI 자동완성이 생산성에 큰 기여를 한다면 GRAM을 쓸 이유가 없어요. Zed를 Ollama 로컬 모델과 연결해서 쓰는 게 훨씬 낫죠. 비용도 줄이고 외부 API 전송도 줄일 수 있어요.

반면 터미널+Neovim 조합에서 GUI 에디터로 막 넘어오는 개발자, 또는 포커스가 필요한 작업 중에 AI 알림이 방해된다고 느끼는 개발자라면 GRAM이 맞는 선택이에요.

**지금 당장 해볼 수 있는 것:**
- Zed를 먼저 AI 없는 모드로 써보세요 (로그인 없이 사용 가능해요)
- GRAM GitHub에서 최신 빌드를 받아 macOS/Linux에서 일주일 테스트해보는 게 좋아요

금융, 의료, 공공기관 개발팀이라면 GRAM 같은 오픈소스 포크를 내부 빌드로 관리하는 방식이 현실적인 선택지예요. Zed 코드베이스가 전부 오픈소스니까 내부 보안팀이 코드를 직접 검증할 수 있어요.

## 앞으로 어떻게 될까요

GRAM과 AI 없는 Zed 포크는 단순한 반AI 정서가 아니에요. "나는 AI가 필요 없는데 왜 AI 중심 구조를 강요받아야 하나"라는 구조적 불만이에요.

짚어둘 세 가지가 있어요. Zed가 AI-optional 모드를 더 명확히 지원하면 GRAM의 존재 이유가 약해져요. 반대로 Zed가 AI를 더 깊이 통합할수록 포크 생태계는 더 강해지죠. 그리고 로컬 LLM 성능이 올라올수록 "Zed + Ollama" 조합이 GRAM보다 나은 답이 될 수도 있어요.

지금 팀에서 쓰는 에디터, AI 기능 때문에 선택했나요—아니면 어쩌다 보니 기본값이 됐나요? 그 질문에 대한 답이 GRAM을 쓸 이유가 있는지 없는지를 갈라요.
