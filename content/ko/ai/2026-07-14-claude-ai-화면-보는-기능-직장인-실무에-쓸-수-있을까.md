AI가 내 Mac 화면을 직접 보고, 클릭하고, 파일을 열고 저장한다. 공상과학 얘기가 아니에요. 2026년 7월 현재, Anthropic의 Claude가 실제로 이 일을 하고 있거든요.

그런데 현실적인 질문이 하나 생겨요. "직장인 실무에 실제로 쓸 수 있을까?" 기능이 있다는 건 알겠는데, 내 업무에 붙이기엔 아직 이른 건지, 아니면 지금 당장 Pro 구독권을 사야 하는 건지 — 이게 궁금한 거잖아요.

데이터를 보면 방향이 보여요. Anthropic이 공개한 사용 통계에 따르면 Claude Cowork 사용의 90% 이상이 코딩이 아닌 일반 업무예요. 업무 프로세스·운영 33.4%, 콘텐츠 제작 16.4%, 소프트웨어 개발 8.7% 순이죠. 개발자보다 기획자·마케터·운영 담당자가 더 많이 쓴다는 뜻이에요.

## 이 기능이 어디서 왔는지: 타임라인

Anthropic이 Claude AI 화면 보는 기능을 처음 공개한 건 2025년 3월 23일이에요. "Computer Use 리서치 프리뷰"라는 이름으로, macOS에서 마우스를 직접 움직이고 클릭하며 앱을 넘나드는 걸 선보였죠.

그 후 흐름이 빠르게 붙었어요.

- **2026년 1월**: Cowork를 macOS 정식 기능으로 출시 (리서치 프리뷰 딱지 제거)
- **2026년 2월**: Windows 지원 추가
- **2026년 3월**: Max·Team·Enterprise 플랜에 100만 토큰 컨텍스트 창 기본 제공, Memory 기능 전 플랜 확대
- **2026년 7월 7-8일**: Cowork를 모바일(iPhone, iPad, Android)과 웹(claude.ai)으로 확장 — Max 구독자 우선 베타 롤아웃

모바일 확장이 왜 지금 중요하냐면, 이전까지는 Mac 앞에 앉아 있어야만 지시를 내릴 수 있었거든요. 이제는 회의 중에 폰으로 "저 보고서 마감 전에 정리해줘"라고 보내면, Mac에서 작업이 돌아가고 완료 알림이 폰으로 와요. 내가 자리를 비워도 컴퓨터가 일하는 구조가 된 거예요.

경쟁 지형도 빠르게 바뀌고 있어요. Meta는 스타트업 Manus를 인수해 월 $17-20 수준의 로컬 파일·앱 제어 데스크탑 에이전트를 내놨고, OpenAI는 "Operator"의 제한적 베타를 돌리며 2026년 본격 출시를 준비 중이에요. Google은 ChromeOS 기반 브라우저 에이전트를 실험하고 있고요. 직장인 실무 적용을 고민하는 동안, 시장 자체가 이미 이 방향으로 굳어지고 있는 셈이에요.

## 실제로 어떻게 작동하나: 세 가지 핵심 포인트

### 권한 구조: 생각보다 보수적이에요

Claude AI 화면 보는 기능이 "내 Mac을 맘대로 조작한다"는 건 과장이에요. 실제 구조는 꽤 신중해요.

Cowork는 폴더 단위 접근 권한만 부여받아요. "Documents/보고서" 폴더 접근을 허용하면, 그 범위 안에서만 파일을 열고 수정해요. 각 액션 단계마다 명시적 승인을 요청하는 옵션도 있고요.

처리는 로컬 Mac에서 이뤄져요. 클라우드 서버로 화면이 스트리밍되는 게 아니라, Mac이 깨어 있고 인터넷에 연결된 상태에서 로컬 실행돼요. 회사 보안 정책이 까다로운 곳이라면 이 부분이 도입 허들이 될 수 있어요.

### 성공률 50%: 어떤 작업에서 갈리나

MacStories의 테스트 결과는 냉정해요. 전반적 성공률 약 50%. 그런데 이게 작업 종류에 따라 크게 달라져요.

**잘 되는 것**: 이메일 요약, 단일 앱 내 반복 작업, 텍스트 파일 정리, 정해진 양식에 데이터 채우기

**자주 실패하는 것**: Safari 여러 탭 넘나들기, Terminal 세션 제어, 두 개 이상의 앱을 동시에 조작하는 복합 작업

단일 앱에서 패턴이 명확한 작업은 안정적이에요. 화면 흐름이 복잡해질수록 성공률이 떨어지고요.

### 모바일 확장: Dispatch 기능이 핵심

2026년 3월 17일 출시된 Dispatch 기능과 7월의 모바일 확장이 묶이면서 사용 방식이 바뀌었어요. Mac에 QR코드로 폰을 페어링하면, 폰에서 보낸 메시지가 Mac의 Cowork 에이전트로 전달돼요. 장시간 작업이 완료되면 폰으로 알림이 와요.

백그라운드 실행도 돼요. 내 모든 기기가 오프라인 상태여도 작업이 계속 돌아가는 클라우드 기반 자율 실행 모드예요. 예약 작업(매일 특정 시간에 보고서 생성 등)은 Pro 이상 모든 유료 플랜에서 쓸 수 있어요.

## 경쟁 비교: 지금 선택지 세 개

| 항목 | Claude Cowork | Meta Manus | OpenAI Operator |
|------|---------------|------------|-----------------|
| 가격 | Pro $20/월 (기본), Max $100-200/월 | $17-20/월 | 베타 (가격 미공개) |
| 화면 제어 | macOS 로컬 실행 | 로컬 파일·앱 제어 | 브라우저 기반 |
| 모바일 연동 | 2026년 7월 확장 (Max 우선) | 미공개 | 미공개 |

Claude의 강점은 투명한 사용 데이터예요. 실제 어떤 업무에 쓰이는지 수치를 공개한 건 Anthropic이 유일해요. Meta Manus는 GPU 기반 모델 추론을 로컬에서 돌리는 점이 특이하지만, 독립적인 벤치마크 데이터가 아직 없어요. OpenAI Operator는 브라우저 범위에서만 동작해서 네이티브 앱 제어는 불가해요.

지금 시점에서 가장 검증된 선택지는 Claude Cowork예요. 데이터가 있으니까요.

## 직무별 판단: 지금 써도 되는 팀, 기다려야 하는 팀

기능은 분명 있는데, 성숙도가 직무마다 달라요.

**지금 도입해도 되는 경우**

운영·기획 직군이라면 지금 써도 돼요. 반복 보고서 취합, 정해진 양식 데이터 정리, 이메일 분류 — 이런 단일 앱 내 패턴 작업이 Cowork의 강점이에요. Anthropic 데이터의 33.4%가 이 직군에서 나왔다는 게 근거예요.

구체적으로, 매일 아침 전날 데이터를 취합해 보고서 템플릿에 채워넣는 작업을 예약 태스크로 설정하면 돼요. Pro 플랜($20/월)으로 가능해요.

**6개월 더 기다려야 하는 경우**

멀티 플랫폼 리서치가 핵심인 직군은 아직 실패율이 높아요. 경쟁사 분석을 위해 여러 웹사이트를 넘나들거나, 브라우저와 Excel과 슬랙을 동시에 오가야 하는 작업이 여기 해당해요. MacStories 테스트에서 Safari 멀티탭 작업이 자주 실패한 게 정확히 이 케이스예요.

**지금 당장 확인해야 할 것**: 회사 보안 정책이에요. 로컬 실행이라 해도 폴더 접근 권한을 부여하는 구조는 IT 부서 승인이 필요할 수 있어요. 실제 도입에서 가장 많이 막히는 지점이 여기예요.

## 앞으로 6개월: 무엇을 지켜봐야 하나

지금 Claude AI 화면 보는 기능의 위치를 정리하면 이래요.

- **지금 된다**: 단순 반복 업무, 단일 앱 제어, 예약 태스크
- **곧 된다 (3-6개월)**: 멀티탭 안정화, Windows Dispatch 연동 완성
- **아직 멀다**: 복잡한 크로스앱 자율 실행

향후 6개월 안에 주시할 신호 세 가지가 있어요.

첫째, **Windows Cowork의 Mac 수준 완성도 도달 여부**예요. 현재 2월에 Windows를 추가했지만, Computer Use의 macOS 전용 기능은 아직 Windows에 없어요.

둘째, **성공률 공식 데이터 공개 여부**예요. 50%라는 수치는 MacStories 독립 테스트 기준이에요. Anthropic이 내부 벤치마크를 공개하면 실무 적용 기준이 명확해져요.

셋째, **OpenAI Operator의 본격 출시 타이밍**이에요. 경쟁이 붙으면 Claude의 가격이나 기능 출시 속도가 빨라질 가능성이 높아요.

2026년 7월 현재 답은 "일부는 지금, 나머지는 연말"이에요. 그런데 더 현실적인 질문은 따로 있어요. 지금 안 써보면, 이미 쓰고 있는 사람들이 쌓는 운영 노하우를 언제 따라잡을 수 있을까요?
