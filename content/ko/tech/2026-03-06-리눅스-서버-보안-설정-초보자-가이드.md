서버 하나 띄웠더니 바로 이상한 로그가 쌓이기 시작해요. "Failed password for root from..." — 봇이 이미 두드리고 있는 거예요.

실제로 공개 IP를 가진 서버에 SSH 연결 시도가 시작되기까지 평균 몇 분도 안 걸려요. Shodan으로 확인해보면 바로 알 수 있죠. 보안 사고의 80% 이상은 설정 실수나 기본값 방치에서 나온다는 것도 이미 알려진 사실이에요.

이 가이드는 "딱 이것만 해도 된다"는 기준으로 정리했어요.

다 읽고 나면 이런 걸 할 수 있게 돼요:

- SSH 기본 포트 바꾸고 루트 로그인 막기
- UFW로 방화벽 세팅
- Fail2ban으로 무차별 대입 공격 차단
- 불필요한 서비스 끄기

---

> **Key Takeaways**
> - 기본 SSH 포트(22번)를 그대로 두면 하루에도 수천 번 스캔 공격을 받아요. 포트 변경만으로 자동화 공격의 90%를 걸러낼 수 있어요.
> - 루트 계정으로 직접 SSH 로그인하는 건 2026년 기준 가장 위험한 설정 중 하나예요. 반드시 일반 사용자 + sudo 구조로 운영하세요.
> - UFW(Uncomplicated Firewall)는 iptables 래퍼로, 명령 세 줄이면 기본 방화벽을 켤 수 있어요.
> - Fail2ban은 일정 횟수 이상 로그인 실패한 IP를 자동 차단해요. 설치 후 별도 설정 없이도 즉시 작동해요.
> - 정기적인 패키지 업데이트는 보안 설정만큼 중요해요. 자동 업데이트 스크립트를 cron에 등록해두면 놓치지 않아요.

## 지금 당장 챙겨야 하는 이유

클라우드에 VM 하나 띄우면 공개 IP가 생기고, 그 순간부터 봇이 22번 포트를 두드리기 시작해요. AWS든 GCP든 Azure든 똑같아요. CISO 홍석범 님이 정리한 자료에 따르면, 리눅스 서버 보안 사고의 대부분은 기본 설정 방치와 약한 패스워드에서 시작돼요.

**시작 전 준비물**:
- 우분투 22.04 LTS 또는 24.04 LTS 서버 (로컬 VM이나 클라우드 인스턴스)
- sudo 권한이 있는 사용자 계정
- 터미널 기초 명령어 (`cd`, `ls`, `nano` 또는 `vim`)

## 보안 도구 비교: 뭘 써야 해요?

초보자 가이드를 찾다 보면 도구 선택에서 막히는 경우가 많아요. 대표적인 선택지를 정리했어요.

| 항목 | UFW + Fail2ban (이 가이드) | firewalld | iptables 직접 설정 |
|---|---|---|---|
| **난이도** | 낮음 (명령 몇 줄) | 중간 (zone 개념 필요) | 높음 (규칙 직접 작성) |
| **기본 탑재** | 우분투 기본 포함 | RHEL/CentOS 기본 | 모든 배포판 |
| **자동화 지원** | Fail2ban 연동 용이 | 중간 | 별도 스크립트 필요 |
| **유지보수** | 쉬움 | 보통 | 복잡 |
| **추천 대상** | 초보자·스타트업 | RHEL 계열 사용자 | 고급 사용자 |

UFW는 iptables를 감싼 래퍼예요. 내부적으로 iptables를 쓰지만, 명령어가 훨씬 읽기 쉬워서 초보자에게 딱 맞아요. firewalld는 RHEL·Rocky Linux 계열에서 기본으로 쓰는데, zone 개념을 먼저 이해해야 해요. iptables를 직접 건드리는 건 나중에 실력이 쌓이면 도전해보세요.

## 단계별 설정 가이드

### 사전 준비: 일반 사용자 만들기

루트로 직접 작업하지 않아요. 일반 계정을 만들고 sudo 권한을 줘요.

```bash
adduser devuser
usermod -aG sudo devuser
su - devuser
sudo whoami
# 출력: root
```

### Step 1: SSH 포트 바꾸기 + 루트 로그인 막기

```bash
sudo nano /etc/ssh/sshd_config
```

파일 안에서 이 항목들을 찾아서 바꿔요:

```bash
Port 2222
PermitRootLogin no
PasswordAuthentication no
MaxAuthTries 3
```

저장 후 SSH 서비스를 재시작해요:

```bash
sudo systemctl restart sshd
```

> ⚠️ 포트 바꾸기 전에 새 포트를 방화벽에서 먼저 열어두세요. 안 그러면 접속이 끊겨요.

### Step 2: UFW 방화벽 켜기

```bash
sudo apt update && sudo apt install ufw -y
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 2222/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status verbose
```

### Step 3: Fail2ban 설치하기

```bash
sudo apt install fail2ban -y
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
sudo nano /etc/fail2ban/jail.local
```

`[sshd]` 섹션을 찾아서 아래처럼 수정해요:

```ini
[sshd]
enabled = true
port    = 2222
filter  = sshd
logpath = /var/log/auth.log
maxretry = 5
bantime  = 3600
findtime = 600
```

```bash
sudo systemctl enable fail2ban
sudo systemctl start fail2ban
sudo fail2ban-client status sshd
```

### Step 4: 자동 보안 업데이트 설정

```bash
sudo apt install unattended-upgrades -y
sudo dpkg-reconfigure --priority=low unattended-upgrades
```

## 실전 예시: 설정 상태 한눈에 점검하기

서버에 처음 붙었을 때 현재 열린 포트와 실행 중인 서비스를 빠르게 확인하는 스크립트예요.

```bash
#!/bin/bash
echo "=== 열린 포트 목록 ==="
ss -tlnp
echo ""
echo "=== UFW 방화벽 상태 ==="
ufw status numbered
echo ""
echo "=== Fail2ban 차단 현황 ==="
fail2ban-client status sshd 2>/dev/null || echo "Fail2ban 미설치"
echo ""
echo "=== 마지막 SSH 로그인 시도 (실패) ==="
grep "Failed password" /var/log/auth.log | tail -10
```

```bash
chmod +x check_security.sh
sudo ./check_security.sh
```

## 이런 경우엔 안 통해요

설정을 다 했다고 끝이 아니에요. 이 가이드만으로 막기 어려운 상황도 있어요.

- **공개키 없이 패스워드 인증만 끄면**: 자기가 접속 못 하게 되는 상황이 생겨요. 공개키 등록 먼저 확인하세요.
- **UFW 켜기 전에 SSH 포트 안 열면**: 락아웃(lockout) 당해요. 포트 먼저 열고 UFW를 켜세요.
- **`jail.conf` 직접 수정하면**: 업데이트 시 덮어써져요. `jail.local`을 만들어서 수정하세요.
- **Fail2ban만으로 제로데이 취약점은 막을 수 없어요**: 패키지 업데이트를 꾸준히 해야 하는 이유가 여기 있어요.

### 체크리스트

- [ ] 루트 SSH 로그인 비활성화 완료
- [ ] SSH 포트 기본값(22)에서 변경 완료
- [ ] UFW 활성화 및 필요한 포트만 개방
- [ ] Fail2ban 설치 및 sshd 감시 활성화
- [ ] 자동 보안 업데이트 등록 완료
