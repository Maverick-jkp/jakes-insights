> **핵심 요약**
> - 개발 환경 세팅 서비스는 Upwork 기준 시급 $45–$85(약 6만–11만원)로 책정되고, 건당 $100–$300(약 13만–40만원) 고정 요금제도 흔해요.
> - 수요는 실재해요. 스타트업 창업자, 디자이너 출신 개발자, 비개발 팀장 — 이 사람들은 Docker, Node 버전 충돌, WSL 세팅을 스스로 해결할 시간이 없거든요.
> - 첫 수익까지는 2–3주. 하지만 첫 달은 거의 세팅과 샘플 작업 위주고, 월 50만–150만원 구간이 현실적으로 나오는 건 2–3개월 이후예요.
> - 한국에서 진입 장벽은 언어보다 포트폴리오예요. 실제 "세팅해준 증거"가 없으면 플랫폼에서 클릭도 안 받아요.

미국 Upwork에서 "development environment setup"으로 검색하면 지금 이 순간 활성 공고가 200개 넘게 떠 있어요. 건당 $150–$400짜리 의뢰가요. 소요 시간은 보통 2–4시간. 시급으로 환산하면 최소 $75, 많게는 $150에 달해요. 이 일을 한국 개발자 대부분이 "너무 쉬운 일"이라는 이유로 그냥 지나쳐요. 그게 진짜 문제예요.

---

## 실제로 어떤 사람이 의뢰하는가

개발 환경 세팅이 "쉽다"고 느끼는 건 경력자 시선이에요. 시장을 보면 달라요.

**주요 클라이언트 유형 세 가지:**

- **스타트업 비개발 창업자:** Python 환경 만들다가 pip 충돌, virtualenv 설정에서 멈춘 사람들이에요. 기술 이해는 있지만 손을 못 써요. Upwork에서 $100–$200짜리 고정 견적 의뢰를 자주 올려요.
- **디자이너/기획자 출신 주니어 개발자:** Figma에서 React로 넘어가려는데 Node 버전 때문에 첫날부터 막히는 케이스예요. Fiverr에서 $50–$150짜리 세팅 요청이 많아요.
- **해외 중소기업 팀장:** 새 개발자 온보딩할 때마다 환경 세팅 문서화가 없어서 반복 작업이 생기는 경우예요. 이건 $300–$600짜리 "팀 개발 환경 세팅 + 문서화" 패키지로 팔 수 있어요.

세 유형 모두 공통점이 있어요. 구글링할 시간보다 돈이 더 빠른 사람들이에요. 그리고 이 수요는 2026년에도 줄지 않았어요. AI 도구가 많아져도 실제로 실행하고 오류 잡아주는 사람은 여전히 필요하거든요.

## 실제 서비스 구성과 단가

막연하게 "세팅 도와드립니다"로는 안 팔려요. 패키지를 구체적으로 쪼개야 해요.

**Fiverr 기준 현실 단가 (2026년 4월 기준):**

| 서비스 | 가격 범위 | 소요 시간 |
|--------|-----------|-----------|
| Mac/Windows 개발 환경 세팅 (Node + Git + VS Code) | $50–$100 | 1.5–2시간 |
| Docker + Docker Compose 환경 구축 | $80–$150 | 2–3시간 |
| Python/Django 로컬 개발 환경 | $70–$130 | 2–3시간 |
| 팀 온보딩 환경 + README 문서화 | $250–$500 | 5–8시간 |

**Upwork에서는 시급 계약이 더 유리해요.** 초반에는 $35–$50/hr로 시작해도 리뷰 5개 쌓이면 $65–$85/hr로 올릴 수 있어요. 첫 달에 5건 완료하면 월 40만–80만원, 리뷰가 붙은 이후 2–3개월차에는 월 100만–200만원도 가능해요.

솔직히 말하면, Fiverr는 처음 진입에 유리하고, Upwork는 단가를 끌어올리는 데 유리해요. 둘 다 동시에 운영하는 게 베스트예요.

## 한국에서 실제로 부딪히는 장벽

이론은 좋은데. 한국 개발자 입장에서 현실을 보면요.

**장벽 1 — 포트폴리오가 없어요**
Fiverr, Upwork 모두 "증거 없으면 안 팔린다"는 구조예요. 그래서 첫 2주는 샘플 작업을 만드는 데 써야 해요. Mac M1 + Homebrew + Node 20 + pnpm 세팅 스크린샷, Docker Compose 실행 화면, README 파일 — 이 세 가지만 있어도 프로필이 살아나요.

**장벽 2 — 영어 커뮤니케이션**
클라이언트와 채팅으로 소통해야 해요. 근데 사실 개발 환경 세팅 의뢰는 기술 용어가 대부분이라 영어 난이도가 낮아요. "What OS are you using?", "Can you share a screenshot of the error?" 정도면 커버돼요. 이건 장벽이 아니에요.

**장벽 3 — 세금 처리**
해외 플랫폼 수익은 국내 종합소득세 신고 대상이에요. 연 3.3% 원천징수는 없고 본인이 신고해야 해요. 수익이 월 50만원 이하면 5월 종합소득세 신고 때 기타소득으로 넣으면 돼요. 100만원 이상 꾸준히 나오면 사업소득으로 잡는 게 나아요. 세무사 한 번 상담 비용은 5만–10만원 선이에요.

**장벽 4 — 플랫폼 수수료**
Fiverr는 수익의 20%, Upwork는 첫 $500까지 20%, 이후 10%를 가져가요. 단가 설정할 때 이 수수료 반드시 포함해서 역산해야 해요. $100 의뢰면 실수령은 $80이에요.

## 지루한 중간 과정 — 진짜 어떻게 유지하나

처음 세팅이 끝나면 뭘 해야 할지 막막해지는 시점이 2–4주차예요. 이때 대부분 포기해요.

현실적인 유지 루틴은 이래요. 주 2–3회, 퇴근 후 30분씩 플랫폼 들어가서 새 공고 확인하고 Proposal 보내는 거예요. Upwork 기준 주 5–10개 지원하면 초반엔 응답률이 5–10%예요. 즉 50개 보내야 2–5개 연결돼요. 이 구간이 제일 짜증나는 구간이에요.

그런데 리뷰 5개가 붙는 순간 구조가 바뀌어요. 클라이언트가 먼저 연락을 해오기 시작해요. Upwork의 알고리즘이 검색 상위에 올려주거든요. 이 시점이 보통 3개월차예요.

패키지 상품은 한 번 만들어두면 반복 판매가 돼요. Fiverr에 "Docker 개발 환경 세팅 $99" 상품 하나 올려두면, 잘 팔리는 달에는 4–6건씩 들어오기도 해요. 능동 수입이지만 Proposal 없이 들어오는 구조라는 점에서 반패시브에 가까워요.

Gig를 올리고 나서 할 일은 하나예요. "development environment setup" 키워드로 다른 셀러 10개 프로필을 보면서 내 설명을 업데이트하는 거예요. 이 작업을 이틀 연속 하면, 처음 올렸을 때보다 클릭률이 30–50% 올라가요.
