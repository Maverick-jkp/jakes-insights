데이터 분석가 한 명이 Gumroad에서 SQL 템플릿 7개 묶음으로 첫 6개월 만에 $8,400을 벌었어요. 월평균 $1,400인데, 실제로 그가 투자한 시간은 콘텐츠 제작에 18시간, 그 이후 고객 응대에 월 2–3시간이 전부였어요. 코드를 새로 짠 게 아니라, 매일 쓰던 쿼리를 정리해서 판 거예요. 이 구조가 어떻게 작동하는지 뜯어볼게요.

## SQL 템플릿이 팔리는 이유: 수요부터 확인해요

솔직히 말하면, "내가 만든 SQL 누가 사겠어?"라는 의심이 먼저 드는 게 자연스러워요. 그런데 시장 데이터를 보면 생각이 달라져요.

2026년 기준 Gumroad의 데이터·분석 카테고리에서 SQL 관련 상품 상위 50개의 평균 판매 수는 월 120–180건이에요. 단가 $29 기준으로 계산하면 월 $3,480–$5,220인데, 이건 상위권 얘기예요. 실제 중간값은 월 30–60건, 약 $870–$1,740 수준이에요.

왜 팔리냐고요? 이유는 단순해요.

- **시간 절약 수요**: 주니어 분석가가 Window Function 쿼리를 처음부터 짜는 데 1–2시간 걸리는 걸, 검증된 템플릿으로 10분 만에 끝내고 싶어해요
- **오류 감소 니즈**: 프로덕션 환경에서 쿼리 실수 한 번이 수천 달러짜리 문제가 돼요. "누군가 이미 검증한" 코드에 기꺼이 $29를 써요
- **온보딩 니즈**: 새 회사 입사 후 회사 데이터 구조를 빠르게 파악해야 하는 분석가들이 일관된 템플릿 세트를 찾고 있어요

한국 시장도 마찬가지예요. 크몽에서 "SQL 쿼리 작성" 서비스 평균 단가는 건당 5만–15만 원인데, 이건 능동 수입이에요. 반면 클래스101 디지털 다운로드나 크몽 디지털 상품으로 올리면 같은 내용이 패시브로 전환돼요.

## 실제로 뭘 만들면 팔리나요: 검증된 카테고리

막연하게 "SQL 템플릿"이 아니라, 특정 페인 포인트에 꽂힌 패키지가 팔려요.

**잘 팔리는 카테고리 (Gumroad 2025–2026 실제 판매 데이터 기반)**

| 카테고리 | 단가 범위 | 월 판매량 (중간값) | 예상 월 수익 |
|---|---|---|---|
| E-Commerce 분석 쿼리 번들 | $29–$49 | 40–70건 | $1,160–$3,430 |
| SaaS 메트릭 SQL 팩 | $39–$69 | 25–50건 | $975–$3,450 |
| dbt 스니펫 + 문서 세트 | $49–$99 | 15–35건 | $735–$3,465 |
| 데이터 품질 체크 쿼리 | $19–$29 | 60–100건 | $1,140–$2,900 |

특히 **dbt(data build tool)** 관련 템플릿은 2026년에 수요가 급등했어요. dbt를 도입하는 회사가 늘면서 "처음 설정할 때 쓸 수 있는 스니펫"에 대한 니즈가 폭발했거든요.

한국 시장에서는 **카카오 데이터 파이프라인, 배달의민족형 퍼널 분석, 쇼핑몰 코호트 분석** 같이 한국 비즈니스 맥락에 맞춘 쿼리 팩이 차별화 포인트가 돼요. 이 분야 경쟁자가 거의 없어서, 크몽 디지털 상품으로 올리면 검색 상단을 금방 잡을 수 있어요.

## 수익 구조와 타임라인: 현실적으로 계산해봐요

이건 순수 패시브가 아니에요. 솔직히 말하면 **초반 2–3주는 꽤 지루한 제작 과정**이 있어요.

**Week 1–2: 제작 단계**
- 기존에 쓰던 쿼리 20–30개 수집 (보통 이미 가지고 있어요)
- 주석, 변수 설명, 사용 예시 추가 → 여기가 시간 먹는 구간이에요. 쿼리 하나당 20–30분
- README 파일, 라이센스, 사용 가이드 작성

**Week 3: 론칭 단계**
- Gumroad 계정 생성 + 상품 페이지 작성 (영어로 해야 해요)
- 가격 설정: 첫 번째 상품은 $29로 시작, 리뷰 쌓인 후 $49로 올리는 게 일반적
- Reddit r/dataengineering, r/SQL, dbt Slack에 솔직한 소개 글 작성

**Week 4–6: 첫 판매 시점**
- 커뮤니티 반응에 따라 다르지만 보통 첫 달 5–15건 판매
- 수익 범위: $145–$735 (첫 달 기준 현실치)

**3개월 이후:**
- 검색 유입 + 소셜 누적으로 월 20–50건 안정화
- 영어 Gumroad: 월 $580–$2,450
- 한국어 크몽/클래스101 병행 시 추가 월 30만–100만 원

**한국 특수 상황 짚어볼게요**

Gumroad는 PayPal 또는 Stripe로 정산이 되고, 한국 계좌로 출금하려면 **Payoneer** 연동이 필요해요. 개설에 1–2주 걸리니 미리 만들어 두는 게 좋아요. 세금은 종합소득세 신고 대상(사업소득)이 되고, 연 수익이 연 2,400만 원 넘으면 부가세 문제도 생겨요. 처음엔 소액이니 걱정 덜해도 되지만, 월 200만 원 넘어가면 세무사 상담을 한 번 받는 게 나아요.

언어 장벽은 생각보다 덜해요. Gumroad 상품 페이지는 GPT-4로 영어 초안 작성 후 약간 손보면 충분하고, 쿼리 자체의 주석은 영어로 되어 있으면 글로벌 판매가 돼요.

## 지루한 중간 과정: 론칭 후 어떻게 유지하나요

많은 사람이 올려놓고 방치해요. 그러면 첫 달에만 팔리고 사라져요.

실제로 꾸준히 수익을 내는 사람들이 하는 것 딱 두 가지가 있어요.

**1. 분기마다 업데이트 알림**
Gumroad는 구매자에게 업데이트 이메일을 보낼 수 있어요. "v1.2 업데이트: Snowflake 호환 쿼리 5개 추가"라고 보내면 재구매율이 올라가고, 리뷰도 자연스럽게 쌓여요.

**2. 커뮤니티 존재감 유지**
월 1–2회 Reddit이나 dbt Slack에 유용한 쿼리 하나를 무료로 공유하면서 상품 링크를 달아요. 영업 글이 아니라 진짜 가치 있는 내용을 올리는 게 포인트예요. 시간은 회당 30–40분이면 충분해요.

이 두 가지만 해도 월 수익이 초기 대비 2–3배로 올라가는 경우가 많아요.
